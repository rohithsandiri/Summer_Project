# Security Policy

## Supported Versions

We actively provide security updates for the following versions:

| Version | Supported |
|---------|-----------|
| 1.0.x   | Yes       |
| < 1.0.0 | No        |

---

## Reporting a Vulnerability

We take the security of the Self-Healing Platform seriously. If you find a security vulnerability, please do **NOT** open a public issue. Instead, report it directly to the maintainers:

1. Email the vulnerability details to `rohithsandiri@example.com` (or the repository owner).
2. Include a detailed description of the issue, steps to reproduce, and any proof of concept scripts.
3. We will acknowledge receipt of your report within 24 hours and provide an estimated resolution window (normally within 7 business days).

Thank you for helping keep this project secure!
